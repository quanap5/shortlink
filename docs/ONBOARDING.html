Architecture, full-stack boundaries, AWS service map, deployment runbook, and operational guidance for the ShortLink SaaS MVP.

Python 3.13 FastAPI + Mangum Next.js + Tailwind AWS CDK Cloudflare DNS Serverless MVP

Quick Start

ShortLink is a tenant-scoped SaaS MVP for creating short URLs, redirecting visitors, generating QR codes, and tracking click analytics. It is deployed as a serverless AWS stack with DNS controlled in Cloudflare.

Production app https://link.twinqx.com
Auth domain https://auth.twinqx.com
API Gateway https://6fxgd9257b.execute-api.ap-northeast-2.amazonaws.com
Region ap-northeast-2 for the application stack. CloudFront ACM certificates live in us-east-1.
MVP guardrails: do not add Route53, Kubernetes, Kafka, Redis, EC2, EKS, GraphQL, billing, custom per-tenant domains, or enterprise SSO.

Authentication Flow

The frontend uses Cognito Hosted UI with authorization code flow. User-facing copy hides provider-specific wording. Protected API routes require a Cognito JWT with custom:tenant_id. 

sequenceDiagram
  autonumber
  actor User
  participant FE as Next.js frontend
  participant Auth as Cognito Hosted UI
  participant API as API Gateway
  participant Lambda as FastAPI Lambda
  participant DDB as DynamoDB

  User->>FE: Open /login
  FE->>Auth: Redirect to branded sign in
  User->>Auth: Authenticate
  Auth-->>FE: /auth/callback?code=...
  FE->>Auth: Exchange code for tokens
  Auth-->>FE: ID token with custom:tenant_id
  FE->>API: Protected request with Authorization header
  API->>API: Validate JWT audience and issuer
  API->>Lambda: Forward request
  Lambda->>DDB: Read or write tenant-scoped data
  DDB-->>Lambda: Result
  Lambda-->>FE: JSON response

Create Link Flow

Link creation is protected by tenant auth. API handlers validate request shape and delegate business behavior to LinkCreationService. The service normalizes slugs and tags, validates target URLs, enforces uniqueness, and writes through a repository abstraction.

  autonumber
  actor Admin
  participant UI as Create Link Page
  participant API as POST /links
  participant Service as LinkCreationService
  participant Repo as LinkRepository
  participant DDB as LinksTable

  Admin->>UI: Enter long URL, optional slug, tags, status, expiration
  UI->>API: POST /links with JWT
  API->>Service: create_link(tenant_id, payload)
  Service->>Service: Validate URL, slug, status, redirect type
  Service->>Repo: Check tenant slug and public slug uniqueness
  Repo->>DDB: Query/GetItem
  DDB-->>Repo: Existing or empty
  Service->>Repo: create(Link)
  Repo->>DDB: PutItem
  DDB-->>Repo: Created
  Repo-->>Service: Link
  Service-->>API: Link
  API-->>UI: Link response

Redirect and Analytics Flow

Public redirects do not require a JWT. The redirect route resolves the public slug, publishes a sanitized click event, and returns the configured redirect status. Raw IP and user-agent values are hashed or classified before persistence.

  SQS-->>Worker: Batch event
  Worker->>Events: Store click event
  Worker->>Agg: Increment summary, link, country, device, browser, source

QR Code Flow

QR images are generated dynamically and are not stored. The QR content uses the final short URL, not the long URL. The MVP adds ?src=qr so scans can be counted under source qr.

AWS Service Details

Service Use in ShortLink Senior engineer notes
CloudFront Public entrypoint for link.twinqx.com. Default behavior routes to API Gateway. Frontend path overrides route to S3.
S3 Private bucket for static Next.js export and auth-config.json. Accessed through CloudFront origin access control, not public bucket hosting.
API Gateway HTTP API Routes backend HTTP requests to Lambda. Protected routes use Cognito JWT authorizer. Public routes are health and slug redirect.
Lambda FastAPI backend via Mangum plus SQS click consumer. Backend dependencies are installed from backend/requirements.txt during CDK bundling.
DynamoDB Tenant, link, click event, and aggregate storage. On-demand billing. Current CDK uses destroy removal policy for MVP environment.
SQS Asynchronous click event queue. Keeps redirect path fast and decouples aggregate updates.
Cognito Hosted UI login, registration, JWT claims. Protected APIs require custom:tenant_id. Registration uses a server-only app client secret.
ACM TLS certificates for CloudFront and Cognito custom domain. CloudFront and Cognito custom domain certs must be in us-east-1.
CloudWatch Lambda logs and debugging. Start here for runtime errors, auth failures, and SQS consumer issues.

Security Model

Tenant isolation Protected APIs derive tenant_id from Cognito claims. DynamoDB tenant-owned tables are keyed by tenant_id.
Auth Cognito Hosted UI handles credentials. Frontend stores tokens for authenticated API calls. API Gateway validates JWTs before Lambda.
Redirect privacy Redirect analytics hash visitor identifiers and do not store raw IP addresses or raw user-agent strings.
URL safety Link creation rejects non-http schemes, localhost, private IP ranges, and internal hostnames.
QR safety QR images are generated dynamically from tenant-scoped links and are not persisted.
DNS boundary DNS is Cloudflare-managed. Do not create Route53 resources.

  • AWS CLI configured for account 373249432962.

Senior Engineer Handoff Checklist

  1. Read AGENTS.md and keep the MVP exclusions intact.
  2. Review infra/lib/shortlink-stack.ts before changing AWS resources.
  3. Keep API handlers thin; place business behavior in services.
  4. Add tests for every feature or bug fix.
  5. Run backend pytest and ruff before deploy.
  6. Run frontend tests, lint, and build before deploy.
  7. Deploy with CDK only after frontend/out is freshly built.
  8. Do production smoke checks after every deploy.
  9. Do not put raw IP addresses or raw user-agent strings in persistent analytics.
  10. Do not add Route53, Redis, Kafka, Kubernetes, EC2, EKS, GraphQL, or microservices.
