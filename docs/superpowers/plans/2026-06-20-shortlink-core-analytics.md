# ShortLink Core Analytics Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Revise ShortLink so the MVP clearly supports shortening long URLs, redirecting short URLs, and showing analytics including hits, device/browser hints, and coarse geolocation.

**Architecture:** Keep the current FastAPI + Mangum + DynamoDB + SQS + Cognito structure. Redirect requests publish click events to SQS; a separate Lambda consumer persists normalized click events and updates tenant-scoped analytics aggregates. The frontend reads protected analytics APIs using the existing Cognito access token.

**Tech Stack:** Python 3.13, FastAPI, Pydantic v2, Mangum, DynamoDB, SQS, AWS Lambda, AWS CDK, Next.js, TypeScript, Tailwind, pytest, ruff.

---

## Current State

- `POST /links` creates tenant-scoped links with a user-provided slug.
- `GET /{slug}` resolves a link and publishes a click event through SQS.
- `GET /links` lists links for the authenticated tenant.
- `ClickEventsTable` exists, but there is no SQS consumer Lambda.
- Analytics frontend is still a placeholder.
- Short URLs generated by frontend currently use the API Gateway base URL, not `https://link.twinqx.com/<slug>`.

## Target MVP Behavior

- A user can shorten a long URL with either a custom slug or generated slug.
- A visitor can access a short URL and be redirected to the original URL.
- Each redirect records a click event asynchronously.
- Analytics show:
  - total hits per link
  - recent click events
  - device family derived from `User-Agent`
  - browser family derived from `User-Agent`
  - coarse country code derived from CloudFront/API Gateway headers when available
- No billing, enterprise SSO, Redis, Kafka, Kubernetes, GraphQL, EC2, EKS, Route53, or microservices.

## Risks And Notes

- Geolocation should be coarse only. Do not store precise IP-derived location in MVP.
- If storing IP address, consider hashing or omitting it before production.
- `link.twinqx.com/<slug>` conflicts with frontend routes unless routing is designed carefully.
- CloudFront viewer country headers are available when traffic enters through CloudFront; API Gateway direct calls may not include them.
- Keep analytics eventually consistent because redirect should stay fast.
