ShortLink SaaS MVP Architecture

Full serverless architecture for the ShortLink MVP, based on docs/SRS.md, AGENTS.md, and the current AWS CDK stack. DNS is managed by Cloudflare, while application workloads run on AWS managed services.

FastAPI + Mangum Next.js + TypeScript + Tailwind AWS CDK Cloudflare DNS No Route53 No Redis/Kafka/Kubernetes

1. System Context

Users access the dashboard and short-link redirects through Cloudflare DNS records that point to AWS origins. CloudFront serves the frontend and forwards API paths to API Gateway.

3. Frontend and Auth Flow

The Next.js dashboard is a static frontend hosted from S3 through CloudFront. Cognito handles sign-in, and authenticated frontend calls include a JWT for protected API routes. 

sequenceDiagram
  autonumber
  actor Admin as Tenant Admin
  participant CF as CloudFront
  participant S3 as S3 Frontend
  participant FE as Next.js App
  participant Cognito as Cognito Hosted UI
  participant API as API Gateway
  participant Lambda as FastAPI Lambda
  participant DDB as DynamoDB Links

  Admin->>CF: Open dashboard URL
  CF->>S3: Fetch static page/assets
  S3-->>CF: HTML, JS, CSS, auth-config.json
  CF-->>Admin: Dashboard shell
  Admin->>Cognito: Sign in
  Cognito-->>FE: Authorization code callback
  FE->>Cognito: Exchange code for tokens
  Cognito-->>FE: ID/access tokens
  FE->>API: GET/POST /links with Authorization header
  API->>API: Validate JWT with Cognito authorizer
  API->>Lambda: Forward request
  Lambda->>DDB: Query or write tenant-scoped links
  DDB-->>Lambda: Link data
  Lambda-->>API: JSON response
  API-->>FE: Dashboard data

4. Link Redirect and Click Event Flow

Public redirect requests resolve a slug through the backend service. When configured, the backend publishes click events to SQS so redirect latency stays small.

  Visitor->>DNS: Open short URL /{slug}
  DNS-->>Visitor: CloudFront target
  Visitor->>CF: GET /{slug}
  CF->>API: Forward redirect route
  API->>Lambda: Invoke FastAPI route
  Lambda->>LinkService: resolve_redirect(tenant_id, slug)
  LinkService->>LinksRepo: find_by_slug(tenant_id, slug)
  LinksRepo->>DDB: GetItem tenant_id + slug
  DDB-->>LinksRepo: Link target URL
  LinkService->>Publisher: publish click event
  Publisher->>SQS: SendMessage
  LinkService-->>Lambda: Redirect target
  Lambda-->>Visitor: 307/302 redirect
  SQS-.->>ClickWorker: Phase 2 consume message
  ClickWorker-.->>ClickDDB: Persist click event

7. API Surface

GET /health Public health check used by deployments and smoke tests.
GET /links Protected route. Lists tenant-scoped links after Cognito JWT validation.
POST /links Protected route. Creates a short link through the link service.
GET /{slug} Public route. Resolves a slug and redirects to the target URL.

8. MVP Scope Guardrails

Use
  • Cloudflare DNS
  • CloudFront, S3, API Gateway, Lambda
  • DynamoDB on-demand tables
  • Cognito for identity
  • SQS for asynchronous click events
  • CloudWatch structured logs
Do not use for MVP
  • Route53
  • Kubernetes, EKS, or EC2 server hosting
  • Kafka, Redis, GraphQL, or microservices
  • Multi-region routing or Global Accelerator
  • Billing, custom domains, or enterprise SSO
Quality rules
  • Every feature includes tests
  • No business logic inside API route handlers
  • Repository/service/domain structure
  • Type hints and structured logging
  • pytest, ruff check ., frontend tests and lint
